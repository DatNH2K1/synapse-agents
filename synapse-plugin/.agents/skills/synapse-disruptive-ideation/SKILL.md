# Disruptive Ideation

This skill provides powerful lateral thinking and creative brainstorming frameworks designed to break industry assumptions, dismantle constraints, and spark breakthrough product concepts.

---

## Core Frameworks & Workflows

### 1. First Principles Thinking (FPT)

Dismantle the problem down to its core physical and logical truths, discarding all historical analogies and "how things are normally done."

**Workflow Steps**:

1. **Identify the Assumption**: Write down the traditional belief (e.g., "A taxi service needs to own a fleet of cars").
2. **Deconstruct to Core Truths**: What are the absolute, undeniable facts? (e.g., "People need transportation from A to B," "Many people own idle cars," "Mobile phones can connect them").
3. **Rebuild from Scratch**: Design a completely new solution using ONLY the core truths (e.g., "Create a peer-to-peer platform connecting idle cars with travelers directly").

---

### 2. Worst Possible Idea (WPI)

A reverse-brainstorming framework. By deliberately proposing the most disastrous, illegal, or ridiculous ideas, we eliminate the fear of failure and uncover hidden paths to innovation.

**Workflow Steps**:

1. **Define the Target**: What is the goal? (e.g., "Increase user engagement").
2. **Brainstorm the Absolute Worst Ideas**: Generate ideas that would guarantee failure, crash the server, or alienate everyone (e.g., "Send 100 spam emails an hour," "Delete user data if they don't log in").
3. **Extract the Creative Opposite**: Analyze the worst ideas to find the underlying mechanism and flip it into a brilliant, positive breakthrough (e.g., "Flipping the deletion idea -> Give users a 'Zen mode' where we automatically hide all old/noisy data so their interface is perfectly clean").

---

### 3. ERRC Blue Ocean Grid

A strategic framework to escape competition and create a new value curve.

| Eliminate                                                         | Raise                                                                |
| ----------------------------------------------------------------- | -------------------------------------------------------------------- |
| What industry-standard features should be completely eliminated?  | What elements should be raised well above the industry standard?     |
| **Reduce**                                                        | **Create**                                                           |
| What elements should be reduced well below the industry standard? | What elements should be created that the industry has never offered? |

---

### 4. SCAMPER Framework

A tactical tool to modify existing ideas or products:

- **S**ubstitute: What materials, processes, or rules can we swap?
- **C**ombine: How can we merge this with another product, service, or industry?
- **A**dapt: What other context can this product adapt to?
- **M**odify: Can we magnify, minify, or change its core shape/flow?
- **P**ut to another use: How can a completely different demographic use this?
- **E**liminate: What is the most complex part we can delete?
- **R**everse: What if we did the exact opposite of the current flow?

---

### 5. Sci-Fi Prototyping (SFP)

Extrapolate technology 5 to 10 years into the future. Write a mini narrative of a user interacting with the future product, then scale it back to a practical, modern MVP.

**Workflow Steps**:

1. **Extrapolate**: Imagine future tech (e.g., holographic screens, telepathic inputs, hyper-local AI).
2. **Narrative**: Write a 200-word user story of a customer using this tech seamlessly in their daily life.
3. **Scale to MVP**: What is the closest software equivalent we can build _today_ using APIs, LLMs, and modern frontend tech? (e.g., Replace "telepathic input" with "proactive voice actions").
